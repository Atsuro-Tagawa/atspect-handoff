# システム開発ターミナル

## 1. サブスク接続・請求書発行・2入口整理の設計状況

**サブスク接続（個別カートリンク方式）**
- 完成：実ブラウザでの動作検証、決済テスト手順書、関連の確定案。
- 未着手：代表の管理画面操作（2件）待ち。完了後に決済テストを実施する段階で止まっている。

**内部の区分呼称整理**
- 完成：全関連文書への反映まで完了。未着手なし。

**請求書の電子発行と2入口整理**
- 完成：請求書ひな形（電子送付用）、発行手段の選定、入金消込の手順、2つの申込経路の整理、いずれも設計完了。
- 未着手：申込・打診台帳への項目追加（代表側の作業）待ち。それが終わるまで運用開始できない。カード決済経路を使う場合に限り必要な試走も未実施。

## 2. 制作への実装引き渡しパッケージ

- **渡し済みの新規実装依頼：なし。** サブスク商品ページ（保険用）の部品は、調査の結果すでにテーマ側に実装済みと判明したため、新規の実装依頼は発生していない。
- **未着手（文書としては作成済み・引き渡しはこれから）：** 商品ページ側の確認作業を制作Tに依頼する文書は用意できているが、その前提となる代表側の管理画面操作がまだのため、制作Tへの実質的な作業引き渡しはまだ行っていない。
